# 📐 Наклонки: наклонные уровни + график в сигнале

Добавлен отдельный режим `SLOPE_LEVELS_ENABLED` с Telegram-кнопкой **📐 Наклонки** и командами:

- `/naklonki`
- `/slope_levels`
- `/slopes`

## Что делает режим

Когда режим включён, бот пропускает сигнал и авто-сделку только если:

1. базовый сигнал уже прошёл минимальную проходимость;
2. старший тренд совпадает со стороной сделки;
3. для LONG найден наклонный уровень типа **восходящая поддержка**;
4. для SHORT найден наклонный уровень типа **нисходящее сопротивление**;
5. есть нужное число касаний линии;
6. текущая цена находится близко к линии по ATR;
7. вероятность отработки наклонного уровня выше порога.

Если условия совпали, бот:

- повышает итоговую проходимость сигнала;
- ставит сильные наклонки в приоритет за счёт вероятности до `SLOPE_LEVEL_PRIORITY_PROBABILITY`;
- добавляет в основания сигнала параметры уровня;
- прикладывает PNG-график с наклонной линией, entry, SL, TP1 и стрелкой направления;
- открывает авто-сделку только по таким сигналам, если автоторговля включена.

## Основные переменные окружения

```env
SLOPE_LEVELS_ENABLED=false
SLOPE_LEVEL_MIN_BASE_PROBABILITY=75
SLOPE_LEVEL_MIN_LEVEL_PROBABILITY=85
SLOPE_LEVEL_PRIORITY_PROBABILITY=98
SLOPE_LEVEL_LOOKBACK_CANDLES=120
SLOPE_LEVEL_CHART_CANDLES=90
SLOPE_LEVEL_PIVOT_WINDOW=3
SLOPE_LEVEL_MIN_TOUCHES=3
SLOPE_LEVEL_TOUCH_ATR_TOLERANCE=0.45
SLOPE_LEVEL_MAX_ENTRY_ATR_DISTANCE=0.85
SLOPE_LEVEL_MIN_SLOPE_PCT_PER_CANDLE=0.003
SLOPE_LEVEL_TREND_SCORE_ABS=4
SLOPE_LEVEL_PROBABILITY_BONUS=8
SLOPE_LEVEL_SEND_CHARTS=true
```

## Логика вероятности

Вероятность наклонного уровня считается не как гарантированный winrate, а как технический score. В него входят:

- количество касаний;
- точность линии через R²;
- расстояние текущей цены до линии через ATR;
- совпадение со старшим трендом;
- объём;
- подтверждение последней свечой;
- MACD momentum;
- EMA 21/50;
- RSI.

Важно: это фильтр качества сетапа, а не гарантия прибыли. Перед LIVE лучше сначала прогнать PAPER и собрать статистику по закрытым сделкам.
