# ETH Profit v1024 — fix анализа

Что исправлено:

- Новостной фон теперь подключён не только через `CRYPTOPANIC_TOKEN`, но и через бесплатные RSS-ленты.
- Если `CRYPTOPANIC_TOKEN` не задан, бот больше не считает новости автоматически нейтральными, а пробует RSS:
  - Cointelegraph Ethereum RSS
  - CryptoSlate feed
  - CoinDesk RSS
- Можно переопределить RSS-ленты переменной Railway:
  - `ETH_PROFIT_NEWS_RSS_URLS=url1,url2,url3`
- Исправлены почти одинаковые точки LONG/SHORT:
  - LONG-точка теперь принудительно ниже рынка/около поддержки;
  - SHORT-точка теперь принудительно выше рынка/около сопротивления;
  - добавлен минимальный зазор по ATR/цене, чтобы точки не совпадали.
- Исправлено противоречие прогноза:
  - если сценарий SHORT, ожидаемая зона строится ниже рынка по TP выбранного SHORT-плана;
  - если сценарий LONG, зона строится выше рынка;
  - если рынок FLAT, зона строится между поддержкой и сопротивлением.
- Исправлена оценка расходов круга: теперь выводится `total_cost_pct`, а не 0.000% из-за неправильного ключа.
- Версия бота обновлена до `1024`.

Важно: ETH Profit остаётся техническим анализом, а не гарантией прибыли. Проверяй рекомендации малым риском и не увеличивай плечо после убытка.
