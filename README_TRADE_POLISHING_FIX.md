# Trade Polishing Fix — версия 1021

Что добавлено:

- Новая отдельная опция **✨ Полировка торговли** в `/settings` и отдельная кнопка в нижней клавиатуре.
- Вход в PAPER/LIVE запрещается, если потенциальная прибыль до целевого TP не даёт минимум **net RR ≥ 2.0** после учёта:
  - расстояния до SL;
  - расстояний до TP1/TP2/TP3;
  - spread;
  - комиссии входа/выхода;
  - запаса на slippage;
  - funding reserve / funding rate, если биржа отдаёт его через CCXT.
- При включённой полировке бот использует более дальние TP-множители: **TP1 1.2R, TP2 2R, TP3 3R**.
- При включённой полировке авто-сделка закрывается не раньше TP3, чтобы правило 2–3R не конфликтовало с ранним TP1.
- Риск на сделку ограничивается максимумом **1%** от депозита, по умолчанию остаётся **0.5%**.
- После **3 убыточных сделок подряд** авто-входы ставятся на паузу на **12 часов**.
- Лимит открытых авто-сделок увеличен до **3**.
- Добавлен журнал сделок трейдера: `/journal` и кнопка в меню полировки.
- В сделку сохраняются причины входа, версия бота, расчёт RR, издержки, funding, spread, slippage, estimated net PnL.
- Кнопка **🏓 Ping** перенесена в самый низ нижней клавиатуры.
- В `/ping` добавлена строка `Versia bota: 1021`.

Важно:

- Полировка торговли влияет только на авто-входы PAPER/LIVE.
- Ручной анализ монет и отправка сигналов не ломаются.
- Это защитный фильтр, а не гарантия прибыли.
