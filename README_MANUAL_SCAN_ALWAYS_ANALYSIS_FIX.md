# Исправление ручного ввода монеты: анализ + график наклона

Что исправлено:

1. Ручной ввод монеты (`BTC`, `ETH`, `POL`, `BTCUSDT` и т.д.) больше не блокируется настройкой `MIN_SIGNAL_PROBABILITY`.
2. Даже если проходимость ниже порога автоотправки, бот всё равно показывает ручной анализ монеты: LONG/SHORT, процент, вход, SL, TP и причины.
3. Строгие фильтры авто-скана — тренд-фильтр, BTC/ETH-only, AI, улучшения торговли, супер-сделка и режимы наклонок — больше не скрывают ручной анализ. Они только добавляют пояснение, почему авто-сигнал мог бы не пройти.
4. Для ручного ввода всегда строится график с наклонной линией в процентах, если данные свечей получены и доступен matplotlib.
5. Если строгий наклонный уровень найден, график уровня теперь тоже показывает наклон в процентах за окно и за свечу.
6. В тексте ручного анализа заголовок теперь `Анализ монеты`, чтобы это не путалось с авто-сигналом подписчикам.

Авто-скан и авто-торговля остались строгими: пороги и фильтры для автоматической отправки сигналов не отключались.
