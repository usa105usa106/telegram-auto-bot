# Fix: разные версии бота отвечают одновременно

Симптом: на один `/start` приходят ответы с разным порогом — 60%, 80%, 95%.
Это означает, что одновременно запущено несколько процессов с одним `BOT_TOKEN`.

В этой сборке добавлена защита `bot_instance.lock`:
- если уже запущена одна копия бота, вторая копия сразу завершится;
- рабочие настройки не читаются из Railway Variables;
- в Railway должны остаться только `BOT_TOKEN` и `ADMIN_IDS`;
- все настройки меняются через Telegram `/settings` и сохраняются в `/app/data/settings.json`.

Важно: для сохранения настроек и lock-файла после redeploy подключи Railway Volume к пути `/app/data`.

Если старые версии всё ещё отвечают, останови/удали старые Railway services/deployments или перевыпусти токен в BotFather.
